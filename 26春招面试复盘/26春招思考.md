# 春招学习咨询

因为个人家庭和感情原因，想毕业后留在深圳，目标总包可以35w左右。

## 一、本人目前情况

背景：本科湖南工业大学计算机科学与技术，研究生深圳大学计算机技术，都是科班。

实习和秋招上岸：主要技术栈都是 C++ 后端，但是说实话，目前 C++ 基础、后端八股和算法都非常一般，学的都非常不好，都是在简单的使用上面。

### 1.1 实习经历

只有一段20人左右，做工业物联网的小厂实习，用的是go语言，岗位是后端开发工程师；实习主要内容是公司云产品销售系统的后端部分，实习期间没有完整的去实现一个业务线，且只有简单的学了一下go，具体的时间线大致如下。

> - 第一周：因为当时不会go语言，公司大mentor给了我第一周学习go语言的基础语法，熟悉公司开发流程等；
> - 中间两个星期（第一个工作量）：开发公司云产品管理系统的API管理模块、API分组模块，这个API可以简单的理解为整个系统前后端进行通信的数据规范以及后端给前端暴露调用的接口；
> - 最后三个星期（第二个工作量）：解析HTTP请求体里面的action字段，找到后端网关层或者微服务的业务执行方法，具体是怎么做的呢，解析action字段，我们后端将action分成了3类，分别是内部action，系统action和用户自定义的action，内部action主要做前端动态资源的初始化，直接在网关层执行，系统action，比如费用管理，订单管理和商品管理，这些管理业务被拆分成了微服务，需要通过consul进行微服务器的注册与发现，然后将对应的action转发到微服务中执行对应的业务逻辑，用户自定义的action主要是是我们后端给用户配置自云产品的逻辑，用户可以编写LUA脚本发送到后端，在网关层通过Mixgo框架提供的Lua脚本解析器，执行Lua脚本；最后这类Action被分发到网关层或者微服务后，通过通过go语言的反射机制，去找到不同Action对应的方法执行入口。
>
> 所以整体实习期间其实两个月不到。

#### 1.3.3 技术看法

只是一些简单的看法，有点悲观者永远正确的意思，希望老师给一点意见哈哈。

首先，在去年一整年找实习和秋招的过程中，很奇妙地感觉 C++ 和常见的后端业务还是不太搭，导致了只有中大厂和游戏厂有少部分的岗位，因为只有用户量上来了或者特定场景下的极限性能需求，才需要 C++ 进行业务开发，所带来的代价就是极大的开发时间成本，但是呢，很多业务场景其实用 Java 和 Go 做出来也是够用了。

其次，在学习成本上，从我的简单理解上看， C++ 没有像Java和Go一样，有自己的Web框架八股需要背，但是其语言本身呢，没有很强的一套对内存管理约束，给了开发人员操作内存资源的极大权限，所以面试中大厂的时候，对 C++ 可以问的很深，这样导致了 C++ 语言本身的学习成本我认为是比 Java 和 Go 要多很多的，还有一些Linux系统编程和网络编程相关的知识等，都是需要静下心来好好沉淀。

然后也是我本人，不知道为什么从大二第一次接触Java开始，包括去年暑期实习接触Go，我对这种封装性极强的语言，有一种本能的排斥感，虽然我自己对Cpp没有达到热爱主动学习的程度，但是看到Cpp的代码，至少不会有排斥感，内心也一直是希望可以自己以后的工作是基于Cpp语言的，虽然Vibe Coding会让语言之间的跨度门槛越来越低，但是就是本能排斥 Java 和 Go。

最后我也了解到了我云智所在部门的主要业务，主要是做腾讯云CRM的，三轮面试体验都很不错，面试官很愿意和我进行技术上的讨论，而不是一直在问八股之类的，特别是最后一轮面试，技术上我回答错了，反问面试官，面试官都会非常耐心地给出正确思路，所以给我整体感觉可能工作氛围还不错，然后强度是1085，我能接受；但是我后面去了解了这个部门的CRM，发现进去大概率是要转 Go 的，感觉有点难受，因为我还是希望可以学Cpp。

## 二、春招展望

所以基于以上想法和介绍，想向老师请教的就是，我下一步春招，应该选哪一条路线，可以让我在春招能够以较大概率上岸深圳呢，不过这个问题本身就不能回答，因为太宽泛太不确定性了。

> - 基于Java or Go的后端开发；
> - 还是基于 C++ 的后端开发；
> - 我有QT一点点基础，可以尝试走QT客户端吗，记得之前实习面试过字节的客户端开发，也是唯一被问到设计模式的一次，但是我知道设计模式是通用的，不止客户端，只是后端有很多中间件，所以在设计模式这一块基本没被问到过。

然后，基于我目前 C++ 后端技术栈下已做的项目，结合目前LLM基本已经发展的很快很成熟了，包括我面试云智的时候，三轮技术面都问了RAG，MCP的原理及做过基于RAG，MCP的相关应用吗，但是我当时只能回答基本概念，所以我就想着，做一个基于 C++ 后端的AI项目，继续投 C++ 后端的相关岗位，这个选择目前来说是不是最优的呢，亦或者是通过转客户端开发，上岸比较好的公司的概率会大一点吗？

最后我目前的想法就是，其实网上很多人劝退客户端，但是我觉得客户端未尝不是一个好的选择，因为客户端这个赛道的人少一些，竞争会不会小一点呢，我目前对于当前的职业生涯的局部期望就是，能留在深圳，且最好第一年能拿到35w以上，然后公司规模不能太小，对于什么方向，其实因为技术都学的不深，所以我都能接受。再说一下为什么公司规模不能小，因为之前实习体验过小厂的工作氛围了，真的有点不太能接受，感觉没有技术佬带，成长曲线会比较慢，如果看到这里，真是谢谢老师的耐心了，希望可以给一些建议。
